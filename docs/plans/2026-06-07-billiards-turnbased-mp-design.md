# 당구 — 턴제 멀티 (3구 대대 / 4구 중대) 설계서

작성 2026-06-07 · 상태: **설계만**(구현 전) · 대상: game.cocy.io/billiards 전면 재작성

## 1. 현황 & 문제

현재 `billiards/index.html`(2031줄) = **1인용 3구 캐롬 시뮬레이터**.
- 파워 슬라이더 + 당점 캔버스 + 연습 시나리오 + 점수/이닝/최고점.
- 문제: 단일 파일에 물리·UI·시나리오 뒤섞임(유지보수難), 멀티 없음, 규격(대대/중대) 개념 없음, 샷 조작이 "시뮬레이터"식이라 직관성·손맛 부족.

**방향**: 인터페이스 컨셉만 참고. **턴제 멀티 대전**으로 새로, **3구(대대)·4구(중대)** 정식 규격, **두께·스핀 UI/UX**를 핵심으로.

## 2. 규격 (실측 기반)

| 구분 | 플레이면 | 포인트 간격 | 종목 | 공 지름 |
|---|---|---|---|---|
| **대대(국제식)** | 2.84 × 1.42 m | 355.5mm | **3구(쓰리쿠션)** | 61.5mm |
| **중대** | 2.54 × 1.27 m | 306mm | **4구**(+중대 3구) | 65.5mm(국산) |

- 좌표계는 **mm 실측** → 화면은 비율 스케일. 2:1 비율 유지.
- 공/쿠션 물성도 실제 근사값(반발계수·마찰)으로. 대대/중대는 테이블 크기·공 지름만 다르고 엔진 동일.

## 3. 룰

### 3구(쓰리쿠션) — 대대
- 내 수구가 **제1적구 + 제2적구 둘 다 맞히되, 두 번째 적구 맞기 전 쿠션 3회 이상** 접촉 → **1점**, 성공 시 **한 번 더**.
- 실패 → 상대 턴(이닝 종료). 점수 그대로 유지.
- 승리: 목표 점수 선취(핸디캡 가능, 예 10/15/20점) 또는 이닝 제한.

### 4구 — 중대
- 공: 내 수구(흰/노), 빨강 2개, 상대 수구(노/흰).
- 내 수구로 **빨강 2개를 다 맞히면 +1점**(쿠션 조건 없음), 성공 시 한 번 더.
- **상대 수구를 맞히면 실점/감점**(룰 옵션: 1점 차감 또는 0점 이닝종료). 빨강 1개만 맞으면 0점.
- 승리: 목표 점수(예 30/50/100/200…).

### 공통
- **뱅킹(선구 결정)**: 시작 시 헤드스트링에서 동시 쳐 쿠션 왕복 후 단축에 가까운 쪽 선. (멀티 첫 턴 결정)
- **이닝**: 양 플레이어 1회씩 = 1이닝. 애버리지(점수/이닝) 기록.
- **파울**: 공 점프·동시터치·시간초과 → 이닝종료(+4구는 감점 룰).

## 4. 멀티 아키텍처 (server-authoritative)

[[reference_relay_multiplayer]] 패턴 따름 — games-server에 `BilliardsPlugin` 추가, relay 자동배포, 클라는 thin 렌더러.

**핵심 결정: 결정론적 물리 공유 모듈.**
- `physics.js` (순수 함수, 의존성 0) 를 **클라·서버 공용**으로. 같은 입력 → 같은 결과 보장(고정 timestep, 정수/고정소수 누적 주의).
- 샷 액션 = `{ aimAngle, thickness(또는 ghostPoint), spinX, spinY, power, seed }`.
- **서버**가 physics.js로 시뮬 → 최종 공 위치 + 이벤트열(쿠션·적구 접촉 순서) + 득점/파울 판정 → 권위 상태 갱신·브로드캐스트.
- **클라**는 같은 파라미터로 로컬 재시뮬해 **공 굴러가는 애니메이션** 부드럽게 재생(결과는 서버값이 진실). 좌석=서버 자동(`ai-`)도 그대로.
- 장점: 치트 불가, 관전자·재접속 시 상태 복원, 물리 1벌만 유지.

## 5. 물리 모델 (physics.js)

- **공-공 충돌**: 탄성 충돌(법선 운동량 교환) + **두께=겹침비율**로 분리각 결정. 던짐효과(throw)·마찰 약간 반영.
- **공-쿠션**: 반발계수(쿠션 e≈0.6~0.8) + **사이드 스핀 → 입사/반사각 변형**(회전 먹은 쿠션). 3구의 핵심.
- **구름/마찰**: 미끄럼→구름 전이, 점진 감속(rolling friction). 정지 임계속도.
- **스핀(당점)**:
  - 상(밀어/follow): 충돌 후 전진 가속.
  - 하(끌어/draw): 충돌 후 후퇴(끌림).
  - 좌/우(side): 쿠션 반사각 변화 + 약간의 커브.
- 고정 dt(예 1/240س)로 적분, 모든 공 정지까지 시뮬. 결과 trajectory 샘플(프레임) 반환.

## 6. 샷 UI/UX (★ 핵심)

3축 입력: **방향(조준) · 두께 · 당점(스핀) · 파워**. 모바일 우선, 한 손 조작 가능하게.

### (A) 조준 + 두께 — "유령공" 방식
- 수구에서 뻗는 **조준선** + 제1적구 위에 **유령공(ghost ball)** 표시 → 유령공이 적구와 겹치는 비율 = **두께**.
- 조작: 테이블 드래그로 방향 회전(정밀조준은 화면 가장자리 **미세 다이얼** 또는 두 손가락 확대경).
- 두께 수치 **1/8·1/4·1/3·1/2·3/4·풀**로 스냅 표시(겹침 시각화 + 텍스트). 얇게/두껍게가 한눈에.
- 예상 경로 **첫 구간만** 살짝 표시(너무 다 보여주면 난이도 0 → 옵션: 초급=길게, 고급=짧게/없음).

### (B) 당점(스핀) — 수구 위젯
- 좌하단 **수구 원형 위젯**: 탭/드래그로 당점 지정.
  - 중앙=무회전 / 위=밀어 / 아래=끌어 / 좌·우=좌·우회전 / 대각선 조합.
  - 가장자리 한계(미스큐 영역)는 빨갛게 잠금.
- 현재 당점을 큐대 그래픽 + 수치(예 "끌어 3시 방향")로 피드백.

### (C) 파워 — 당겨치기 제스처
- 수구를 **뒤로 끌었다 놓기**(pull-back) = 파워(끄는 거리=세기), 놓으면 샷. 실제 스트로크 손맛.
- 보조로 하단 파워 슬라이더(정밀/접근성). 파워 게이지 시각화.

### (D) 실행 흐름
1. 내 턴 → 조준선/유령공 드래그로 방향·두께.
2. 당점 위젯으로 스핀.
3. 당겨치기(또는 슬라이더+샷버튼)로 파워→발사.
4. 서버 판정 → 양쪽 애니메이션 동기 재생 → 득점/실점 토스트+음성 → 성공 시 연속, 실패 시 턴 전환.
- "조준 잠금/해제" 토글로 오조준 방지. 되돌리기 없음(친 건 끝).

## 7. 화면 레이아웃 (모바일 세로/가로)

- 상단: 양 플레이어 점수·이닝·애버리지(상대처럼 우측정렬), 종목(3구 대대/4구 중대) 뱃지, 목표점수.
- 중앙: 테이블(가로 회전 권장, 세로도 지원). 공·조준선·유령공·경로.
- 하단: 당점 위젯(좌) · 파워(중) · 샷/확대/설정(우).
- 관전/대기: 상대 차례엔 입력 잠금 + "상대 조준중" 표시.

## 9. 구현 로드맵 (별도 작업)

1. **physics.js** 결정론 엔진 + 단위테스트(공-공/쿠션/스핀 시나리오, 클라·서버 동일결과 검증).
2. 단일플레이 프로토(현 시뮬 대체) — 조준/두께/당점/파워 UI 완성, 3구·4구 룰 판정.
3. games-server `BilliardsPlugin`(registry 등록) + 클라 MultiplayerUI 연동, 좌석 자동.
4. 뱅킹·이닝·애버리지·핸디캡, 랭킹(클라+서버), i18n(한/중/영), 지갑 정산, 음성/효과음.
5. 폴리시: 경로 미리보기 난이도 옵션, 리플레이, 관전.
