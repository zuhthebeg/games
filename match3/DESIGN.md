# 애니팡식 매치-3 게임 — 설계문서

`game.cocy.io/match3` (가칭). 단일 HTML, 우리 공통 스택(지갑·랭킹·i18n·no-cache) 준수. 에셋은 로컬 SDXL 이미지 파이프라인으로 생성.

## 1. 코어 게임플레이 (애니팡 = 시간제 매치-3)
- **보드**: 7×7 그리드(애니팡 표준). 칸마다 6종 블록 중 하나.
- **매치**: 가로/세로 **3개 이상** 같은 블록 일렬 → 제거 + 점수. 인접 두 블록 **스왑**(드래그/탭2회). 스왑은 매치가 생길 때만 유효(아니면 원위치 복귀).
- **낙하/리필**: 제거된 칸 위 블록이 내려오고, 상단에서 새 블록 채움. 연쇄(콤보) 자동 처리.
- **모드**:
  - (A) **시간제** 60초 — 제한시간 안에 최대 점수(애니팡 오리지널). 기본 추천.
  - (B) **이동제한** — N수 안에 목표 점수(캔디크러시식). 옵션.
- **종료**: 시간/이동 소진 → 결과(점수) → 골드 보상 + 랭킹 제출.

## 2. 특수 블록 (콤보 보상)
- 4개 일렬 → **라인 폭탄**(가로/세로 한 줄 제거).
- 5개 일렬(또는 T/L) → **레인보우/같은색 전체 제거**.
- 5초 미스 없으면 콤보 게이지↑(점수 배수). (옵션)
- **셔플**: 가능한 매치 없으면 자동 셔플.

## 3. 점수 / 보상 / 랭킹
- 매치 3=30점, 4=60+폭탄, 5=100+레인보우, 콤보 연쇄마다 ×배수.
- **골드**: 클리어 시 `점수 비례`(예: 점수/10, 상한). 또는 목표달성 보너스. (수치는 협의)
- **랭킹**: `game-rankings.js`에 match3 config + 서버 `functions/api/rankings/match3.ts`(점수형) 추가. best_score 순.

## 4. 에셋 (SDXL 파이프라인 생성)
- **블록 아이콘 6종** (정사각, 투명배경, 512px, 통일된 스타일/외곽선/그림자). 테마는 협의:
  - 후보: ① 귀여운 동물 6종(애니팡식) ② cocy 마스코트/캐릭터 ③ 과일·캔디 ④ 한국풍(화투 오브젝트)
- **특수블록 오버레이**: 라인폭탄(화살표), 레인보우(무지개) — 아이콘 위 합성 or 별도.
- **배경**: 1장(보드 뒤 패턴/그라데이션).
- **이펙트**: 제거 파티클은 CSS/canvas로(이미지 불필요).
- 파이프라인: 동일 프롬프트+시드 고정으로 6종 스타일 통일, app-safe 변형.

## 5. 기술 구조 (단일 HTML)
- 보드 상태 2D 배열, 렌더는 CSS grid + transform 애니(낙하/스왑). 60fps 목표, 이미지 재사용(깜빡임 방지 — 마작 교훈).
- 입력: 포인터 드래그 스왑 + 탭2회 스왑. 터치 우선.
- i18n: 새 게임이므로 **사전식 t()**(ko/zh-TW/en) — 옵저버 아님. 🌐 토글(즉시전환).
- 지갑: `SharedWalletConfig`+`SharedWallet.init()` 필수, 상단 골드바.
- 캐시: no-cache 메타 + lib `?v=` 최신.
- 포털 등록: apps.json(puzzle/arcade), GAME_TW, sitemap, icon.svg.
- (옵션) 효과음: 매치/콤보/폭탄 — 합성 or 추후 더빙.

## 6. 마일스톤
1. **프로토(코어)**: 보드 생성·스왑·매치·낙하·리필·콤보·점수 (플레이스홀더 색블록).
2. **에셋 교체**: 파이프라인 6종 아이콘+배경 적용.
3. **연동**: 지갑·랭킹(클라+서버)·i18n·포털 등록.
4. **폴리시**: 특수블록·이펙트·사운드·튜닝.

## 7. 결정 필요
- 테마(에셋 방향) / 모드(시간제 vs 이동제한) / 보드크기(7×7 기본) / 골드·점수 수치.
