# billiards3d — 물리 리얼리즘 재설계 (설계 문서)

> 상태: **1차 구현 완료(2026-06-08)** — 스트로크(끊어/기본/밀어), 사이드 스쿼트화(경로 직선·쿠션엔 유지), MAX_SPIN 0.7→0.5, FOLLOW_GAIN 1.3→1.0, b3d_tune v2 마이그레이션. 검증: 엔진 14테스트 + 거동 harness(끊어<기본<밀어 / 자유주행 곡률<2.3mm / 쿠션 좌우 반사각 갈림) 통과.
> **남은 단계(2차+):** 충돌 stun 분리각 정교화, throw 실측 캘리브, 스트로크별 접촉시간 모델, 프리셋 해법 재튜닝, 쿠션반사 속도의존 — 아래 6·7절 참고.
> 작성: 2026-06-08. 대상: `game.cocy.io/billiards3d` (`physics.js` 결정론 엔진).
> 트리거(사용자 피드백): "스핀 상하좌우 민감도가 높아서 공의 방향에 영향을 너무 많이 받는다. 실제 당구장 물리법칙과 많이 다르다. 끊어치기·밀어치기 같은 타격에 따른 물리 처리도 필요하다."

---

## 1. 문제 정의 (관측된 증상)

1. **사이드스핀(좌우 당점) 민감도 과다** — 좌우 당점을 주면 수구의 *진행 경로 자체*가 너무 크게 휘거나 방향이 틀어진다. 실제로는 평평한 테이블에서 사이드스핀이 수구의 출발 직선 경로에 주는 영향(스워브/squirt)은 매우 작다.
2. **상하 당점(밀어/끌어) 민감도 과다** — follow/draw가 충돌 전 경로·충돌 후 거동에 과하게 반영됨.
3. **타격(스트로크) 종류 미구현** — 끊어치기(스턴/잽), 밀어치기(밀어 따라가기), 끌어치기(드로)가 당점만으로 뭉뚱그려져 있고, "큐를 어떻게 미느냐"에 따른 차이가 없다.
4. 종합적으로 당점값(`spinX`, `spinY`)이 궤적에 *선형·과대* 반영되어 실제 캐롬 감각과 괴리.

---

## 2. 현재 모델 요약 (`physics.js` 기준, 변경 전)

현재 엔진이 당점을 다루는 방식(세션에서 확인된 구조):

- 수구 상태: 선속도 `(vx, vy)` + 각속도 `wx, wy`(구름축) / `wz`(수직축=사이드).
- 런타임 튜닝 `window.BIL_TUNE` 기본값:
  ```
  TUNE_DEF = { maxpow:7800, cushE:0.90, roll:0.011, follow:1.3, side:1.4, cushSide:0.5, throw:0.045 }
  ```
  - `follow`(FOLLOW_GAIN 1.3): 상하 당점 → 구름 각속도 이득.
  - `side`(SIDE_GAIN 1.4): 좌우 당점 → 수직 각속도 `wz` 이득. `wz = -base * spinX * SIDE_GAIN`.
  - `cushSide`(K_CUSH_SIDE 0.5): 쿠션에서 `wz`가 반사각에 주는 영향.
  - `throw`(0.045): 공-공 던짐(throw).
- 미끄럼→구름 마찰 전이, `wz *= 0.9992`/step 감쇠, 쿠션 시 `CUSH_WZ_KEEP=0.3`.
- 당점 입력 한계 `MAX_SPIN=0.7`, 발사 미스큐 컷 `>0.72`.

**핵심 결함:** `spinX`(사이드)가 `wz`를 통해 수구의 *자유 구름 경로*에 곡률로 직접 반영되는 경로가 과대하고, `spinY`(상하)도 충돌 전 경로/속도에 과하게 섞임. 실제 물리에서 분리되어야 할 효과들이 한 덩어리로 증폭되어 있음.

## 3. 실제 당구 물리와의 차이 (근거)

조사·검증이 필요한 실제 물리 개념(구현 전 수치 캘리브레이션 대상):

### 3.1 당점 → 스핀의 물리적 상한
- 큐팁이 수구 중심에서 벗어난 거리 `b`(반지름 대비 비율). **미스큐 한계 ≈ 0.5R** (접촉점이 반지름의 50%를 넘으면 미끄러져 미스큐). 즉 게임의 `MAX_SPIN=0.7`은 물리적으로 과함 → **0.5 기준**으로 재정의 검토.
- 초기 스핀/속도 비율은 접촉 오프셋에 의해 결정되며 상한이 있다. 무한정 증폭 불가.

### 3.2 사이드스핀은 출발 경로를 거의 안 바꾼다
- **Squirt(deflection):** 사이드를 주면 수구가 조준선에서 *반대쪽으로* 아주 약간(보통 1~3°) 빗나감. 당점이 클수록 커지지만 여전히 작은 각.
- **Swerve(곡선):** 큐를 약간 들고 칠 때만 수구가 곡선. 수평 스트로크 + 평평한 테이블에서는 무시 가능.
- 따라서 **사이드스핀의 주 효과는 (a) 쿠션 반사각, (b) 공-공 throw** 이지 *자유 주행 경로의 큰 곡률이 아님*. → 현재 모델의 가장 큰 괴리 지점.

### 3.3 상하스핀(follow/draw)은 "충돌 후"에 작용
- 충돌 전 수구는 미끄럼→자연구름으로 수렴. 상단 당점=follow(접촉 후 전진), 하단=draw(접촉 후 후퇴), 중앙 부근=stun(접촉 후 정지/수직 분리).
- 핵심: **상하 당점은 적구 충돌 *이후* 수구 거동을 좌우**하고, 충돌 전 직선 경로엔 거의 영향 없음(미끄럼 구간 길이 정도만).

### 3.4 스트로크(타격) 종류 — 본 과제의 핵심
한국식 캐롬에서 "어떻게 미느냐"가 당점만큼 중요:
- **끊어치기 (스턴/잽, short jab):** 짧게 끊어 쳐 follow를 죽임 → 접촉 시점에 가까운 stun 상태. 수구가 분리각으로 깔끔히 갈라짐. 회전 생존이 짧음.
- **밀어치기 (follow-through, push):** 길게 밀어 follow 회전을 *유지/증폭* → 충돌 후 수구가 적구를 따라 더 전진. 회전 생존이 김.
- **끌어치기 (draw):** 하단 + 팔로스루로 후퇴 회전 유지.
- 물리적으로 강체 모델에선 "follow-through"가 초기조건을 직접 못 바꾸지만, 게임에선 **접촉시간/회전 생존(decay)·회전:속도 비율을 스케일하는 스트로크 파라미터**로 모델링하는 것이 실용적.

## 4. 제안 설계

### 4.1 효과 분리 (decoupling) — 가장 중요
당점/스핀이 영향을 주는 채널을 **명시적으로 분리**하고 각각 독립 게인을 둔다:

| 채널 | 입력 | 영향 대상 | 기본 게인(검토) | 현재와 차이 |
|---|---|---|---|---|
| Squirt | spinX | 발사 *직후* 각도 보정(작게) | 작게(예 최대 ~2.5°) | 신규(현재는 경로 곡률로 과대 반영) |
| 자유주행 곡률 | spinX | 수구 자유 주행 휨 | **≈0** (수평 스트로크 가정) | **대폭 축소** |
| 쿠션 반사각 | wz(사이드) | 쿠션 후 진행각 | 유지~소폭↑ | 사이드의 *주* 효과로 이동 |
| 공-공 throw | spinX | 적구/수구 throw | 유지 | — |
| 충돌 후 전진/후퇴 | spinY | 충돌 *후* 수구 vx,vy | 중간 | 충돌 전과 분리 |
| 분리각(stun) | spinY≈0 | 충돌 후 90°-α 분리 | 신규/정교화 | — |

→ 사용자 증상("좌우가 경로에 영향 과다")의 직접 해결: **사이드는 경로가 아니라 쿠션·throw로**.

### 4.2 스트로크 모델
입력에 **스트로크 타입** 추가 (UI: 버튼 3종 또는 슬라이더):
- `stun`(끊어치기): follow 회전 생존 ↓ (decay 빠르게), 회전:속도 비율 ↓.
- `normal`: 기본.
- `follow`(밀어치기): follow 회전 생존 ↑ (decay 느리게), 충돌 후 전진 ↑.
- (옵션) `draw`(끌어치기)는 하단 당점 + follow-through로 표현하거나 별도 타입.

파라미터화:
```
STROKE = {
  stun:   { spinKeep: 0.6, followAfter: 0.3, contact: 'short' },
  normal: { spinKeep: 1.0, followAfter: 1.0, contact: 'mid'   },
  follow: { spinKeep: 1.4, followAfter: 1.6, contact: 'long'  },
}
```
- `spinKeep`: `wx,wy,wz` decay 배율(생존 길이).
- `followAfter`: 적구 충돌 후 수구 전진/후퇴 성분 스케일.

### 4.3 민감도 재캘리브레이션
- `MAX_SPIN` 0.7 → **0.5**(미스큐 한계와 일치), 위젯 눈금도 0.5 기준.
- `SIDE_GAIN`(자유주행 영향) 대폭 ↓, 대신 `CUSH_SIDE`로 이동.
- `FOLLOW_GAIN`은 충돌 후 채널로 한정.
- 모든 게인은 `BIL_TUNE`에 유지하되 **물리적으로 의미 있는 기본값**으로 재설정.

## 5. 조사(리서치) 항목 — 구현 전 확정 필요

1. **미스큐 한계**와 당점오프셋→스핀 비율의 정량 관계(반지름 대비 b/R, 표준 0.5R).
2. **Squirt 각도** vs 당점·큐 무게의 표준 수치(보통 1~3°, "endmass" 의존).
3. **Throw**(공-공) — cut각·상대속도·당점에 따른 throw각(0~수 도).
4. **쿠션 반사** — 입사각·속도·사이드에 따른 반사각 변화(쿠션 압축/스로우 포함) 실측 그래프.
5. 캐롬 **스트로크 별 회전 생존·접촉시간** 정성/정량 자료(끊어/밀어/끌어).
6. 자연구름 전이 거리(미끄럼 마찰계수 μ_slide≈0.2, μ_roll≈0.01 표준).
- 출처 후보: Dr. Dave (Billiards/물리 문서·실험영상), 캐롬 교본, 물리 논문(cue ball deflection/throw).

## 6. 구현 단계(다음 세션)

1. `physics.js` 스핀 채널 분리 리팩터(자유주행 곡률 분리·약화).
2. Squirt를 발사 직후 각보정으로 추가(작게).
3. 충돌 해석에 stun 분리각 + spinY 기반 충돌 후 성분 정교화.
4. 스트로크 타입 파라미터 도입(엔진) + UI(버튼/슬라이더).
5. `MAX_SPIN`/게인 재캘리브레이션 + 당점 위젯 눈금 갱신.
6. 회귀: 기존 프리셋 해법 재검증·재튜닝.

## 7. 검증 시나리오(테스트)

- 사이드만 주고 직진 → 경로가 *거의 직선*(squirt 소각만). 현재는 크게 휨(회귀 기준).
- 끊어치기 정면 → 충돌 후 수구 정지(stun), 적구만 전진.
- 밀어치기 → 충돌 후 수구가 적구 따라 전진.
- 끌어치기 → 충돌 후 수구 후퇴.
- 사이드 + 쿠션 → 반사각이 사이드 방향으로 적절히 꺾임(접시 되돌아오기 재현; 이전 세션 `cushSide` 0.5/0.9992 decay 이슈와 연계).
- 3쿠션 기존 프리셋 해법 6종 여전히 득점.
